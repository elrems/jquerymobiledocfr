Les types de barre d'outils

Dans jQuery Mobile, il y a deux types de barres d'outils standards : Entêtes et pieds de page.

  • La barre d'entête sert de titre de page, est généralement le premier élément de chaque page mobile, et contient généralement un titre de page et jusqu'à deux boutons.
  • La barre de pied de page est généralement le dernier élément à l'intérieur de chaque page mobile, et tend à être plus libre que l'entête en termes de contenu et de fonctionnalité, mais contient généralement une combinaison de texte et de boutons.

Il est très courant d'avoir une navigation horizontale ou une barre d'onglets à l'intérieur de l'entête ou du pied de page. jQuery Mobile inclut un widget navbar qui transforme une liste non ordonnée de lien dans une barre de bouton horizontale, qui fonctionne bien dans ces cas.

Regardez la reference de data- attribute pour voir tous les attributs possibles que vous pouvez ajouter aux barres d'outil.

Options de positionnement de barre d'outils

L'entête et le pied de page peuvent être positionnés sur la page de plusieurs façons différentes. Par défaut, les barres d'outils utilisent le mode de positionnement "inline". Dans ce mode, l'entête et le pied de page se positionnent dans le flux normal du document (le comportement par défaut du HTML), ce qui assure qu'ils sont visibles sur tous les appareils, indépendamment de JavaScript et du support du positionnement CSS.

Le mode de positionnement "fixed" fixe les barres d'outils soit en haut ou en bas de la fenêtre sur les navigateurs qui supportent le CSS de positionnement fixe (qui inclut les navigateurs de bureau, iOS5+, Android 2.2 +, BlackBerry 6 et d'autres). Dans les navigateurs qui ne supportent pas le positionnement fixe, les barres d'outils se replieront vers du statique, le positionnement est intégré dans la page.

Lorsque le basculement est activé, en tapant sur l'écran, cela permet de basculer la visibilité de la barre d'outils fixes. En tapant sur la page, lorsque les barres d'outils ne sont pas visibles, cela les affiche. En tapant de nouveau, cela les cache jusqu'à ce que vous tapiez à nouveau Cela donne aux utilisateurs la possibilité de masquer les barres d'outils pour utiliser le maximum de la surface d'affichage. Un bémol est que les barres d'outils fixes ne sont jamais réellement cachées, mais alternées entre le positionnement fixe et statique. Cela signifie que si vous êtes au sommet d'une page, vous ne pouvez pas basculer une barre d'outils d'entête en dehors de l'affichage, car l'endroit pour basculer dans le flux du document est le haut de la page. C'est la même chose pour les pieds de page fixes lorsqu'on se trouve au bas de document.

Pour définir ce comportement sur une entête ou un pied de page, ajoutez l'attribut data-position="fixed" à l'élément entête ou pied de page.

Le mode de positionnement "fullscreen" fonctionne exactement comme le mode fixe, sauf que les barres d'outils se superposent sur le contenu de la page au lieu de se réserver une place dans le document. Ceci est utile pour des applications d'immersion comme des aperçus de photo ou de vidéo où vous voulez que le contenu remplisse tout l'écran et où les barres d'outils peuvent être citées à comparaître en touchant l'écran. Gardez à l'esprit que les barres d'outils dans ce mode se positionneront au dessus du contenu de la page c'est donc la meilleur utilisation pour des situations spécifiques.
