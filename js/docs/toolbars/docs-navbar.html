Barre de navigation simple

jQuery Mobile a un widget de barre de navigation très simple qui est utile pour fournir un maximum de 5 boutons avec des icônes en option dans une barre, généralement au sein d'une entête ou d'un pied de page. Il y a également une variante, une barre de navigation persistante qui fonctionne davantage comme une barre d'onglets qui reste fixe pendant que vous naviguez à travers les pages.

Une barre de navigation est codée comme une liste non ordonnée de liens enveloppée dans un élément de conteneur qui contient l'attribut data-role="navbar". C'est un exemple avec une barre de navigation avec deux boutons :


<div data-role="navbar">
	<ul>
		<li><a href="a.html">Un</a></li>
		<li><a href="b.html">Deux</a></li>
	</ul>
</div><!-- /navbar -->

Quand un lien dans la barre de bavigation est cliqué, il porte le statut actif (selected). La classe ui-btn-active est d'abord supprimée de toutes les ancres dans la barre de navigation avant d'être ajouté au lien activé. Si c'est un lien vers une autre page, la classe sera à nouveau supprimée après que la transition soit terminée.

Pour qu'un élément soit au statut actif avant l'initialisation de la barre de navigation, ajoutez class="ui-btn-active" à l'ancre correspondant dans votre balisage. Additionnellement, vous pouvez ajouter une classe ui-state-persist de façon que le framework restaure l'état actif à chaque fois que la page sera affichée. L'exemple ci-dessous montre une barre de navigation avec l'élément "Un" définie à actif :


<div data-role="navbar">
	<ul>
		<li><a href="a.html" class="ui-btn-active ui-state-persist">Un</a></li>
		<li><a href="b.html">Deux</a></li>
	</ul>
</div><!-- /navbar -->

Les éléments de la barre de navigation sont définis pour diviser l'espace uniformément dans ce cas, chaque bouton est la moitié de largeur de la fenêtre du navigateur:

  • Un
  • Deux

L'ajout d'un troisième élément rendra automatiquement chaque bouton avec un tiers de la largeur de la fenêtre du navigateur :

  • Un
  • Deux
  • Trois

L'ajout d'un quatrième élément rendra automatiquement chaque bouton avec un quart de la largeur de la fenêtre du navigateur :

  • Un
  • Deux
  • Trois
  • Quatre

La barre de navigation autorise 5 éléments au maximum, chacun utilise un cinquième de la largeur de la fenêtre du navigateur :

  • Un
  • Deux
  • Trois
  • Quatre
  • Cinq

Si plus de 5 éléments sont ajoutés, la barre de navigation sera tout simplement envelopper sur plusieurs lignes :

  • Un
  • Deux
  • Trois
  • Quatre
  • Cinq
  • Six
  • Sept
  • Huit
  • Neuf
  • Dix

Les barres de navigation avec 1 élément sera simplement rendu à 100%.

  • Un

Barres de navigation dans les entêtes

Si vous souhaitez ajouter une barre de navigation en haut de la page, vous pouvez toujours avoir un titre de page et des boutons. Il suffit d'ajouter le conteneur de la barre de navigation à l'intérieur du bloc de l'entête, juste après le titre et les boutons dans l'ordre du code.

Barres de navigation dans les pied de page

Si vous souhaitez ajouter une barre de navigation au bas de la page afin qu'elle agisse plus comme une barre d'onglets, il suffit d'envelopper la la barre de navigation dans un conteneur avec un data-role="footer"


<div data-role="footer">		
	<div data-role="navbar">
		<ul>
			<li><a href="#">Un</a></li>
			<li><a href="#">Deux</a></li>
			<li><a href="#">Trois</a></li>
		</ul>
	</div><!-- /navbar -->
</div><!-- /footer -->
  • Un
  • Deux
  • Trois

Icônes dans les barres de navigation

Les icônes peuvent être ajoutées aux éléments de la barre de navigation en ajoutant l'attribut data-icon en spécifiant une icône mobile standard pour chaque ancrage. Par défaut, les icônes sont ajoutées au-dessus du texte (data-iconpos="top"). Les exemples suivants ajoutent des icônes à la barre de navigation dans un pied de page.

La position de l'icône est définie sur le conteneur de la barre de navigation plutôt que cela soit défini individuellement sur les liens par souci de cohérence visuelle. Par exemple, pour placer les icônes dessous les textes, ajoutez l'attribut data-iconpos="bottom" au conteneur de la la barre de navigation.


<div data-role="navbar" data-iconpos="bottom">

Thématisation des barres de navigations

Les barres de navigation héritent du nuancier de thème de leur conteneur parent, tout comme les boutons. Si une la barre de navigation est placée dans l'entête ou le pied de page de la barre d'outils, elle héritera du nuancier "a" de la barre d'outils par défaut pour les barres, sauf si vous définissez ceci dans le balisage.

Voici quelques exemples de barres de navigation dans différents nuancier de conteneur qui automatiquement héritent de la lettre du nuancier de leurs parents. Notez que dans ces exemples, au lieu d'utiliser un attribut data-theme, nous ajoutons manuellement les classes de nuancier pour appliquer le nuancier du corps (ui-body-a) et la classe pour ajouter l'espacement standard du corps (ui-body), mais l'héritage fonctionne de la même manière :

  • B
  • C
  • D
  • E

Nuancier "d"

  • A
  • B
  • C
  • D
  • E

Pour définir la couleur du thème pour un élément de la barre de navigation, ajoutez l'attribut data-theme pour les liens individuels et spécifiez un nuancier du thème. Notez que l'application d'un nuancier de thème au conteneur de la barre de navigation n'est pas supporté.
