Le menu de sélection est basée sur un élément select natif, qui est caché de la vue et remplacé par un bouton de sélection avec un style personnalisé qui correspond à l'aspect et à la convivialité du framework jQuery Mobile. Le menu de sélection est ARIA-enabled et accessible par le clavier sur un ordinateur de bureau. Regardez la reference de data- attribute pour voir tous les attributs possibles que vous pouvez ajouter aux sélections.

Par défaut, le framework s'appuie sur les options du menu natif de l'OS pour l'utiliser avec le bouton personnalisé. Lorsque le bouton est cliqué, le menu natif de l'OS va s'ouvrir. Quand une valeur est sélectionnée et que le menu se ferme, le texte du bouton personnalisé est mis à jour pour correspondre à la valeur sélectionnée. Veuillez noter que le framework offre aussi la possibilité d'avoir des menus de sélection personnalisés (non natif), voir les détails au bas de cette page et dans la page menu de sélection personnalisé.

Pour ajouter un menu de sélection sur votre page, commencez avec un élément standard select peuplé avec un ensemble d'élément option. Réglez l'attribut for de label pour faire correspondre l'id du select ainsi ils sont sémantiquement liés. Il est possible de cacher le label d'accessibilité si il n'est pas souhaitée dans la mise en page, mais nous avons besoin qu'il soit présent dans le balisage pour des raisons de sémantique et d'accessibilité.

Le framework trouvera tous les éléments select et les améliorera automatiquement dans des menus de sélection, pas besoin d'appliquer l'attribut data-role. Pour éviter l'amélioration automatique d'un select, ajouter l'attribut data-role="none" au select.


<label for="select-choice-0" class="select">Mode de livraison :</label>
<select name="select-choice-0" id="select-choice-0">
   <option value="standard">Lent : 7 jours</option>
   <option value="rush">Rapide : 3 jours</option>
   <option value="express">Express : le lendemain</option>
   <option value="overnight">De nuit</option>
</select>

Cela va produire un menu de sélection de base. Les styles par défaut définissent la largeur de champ de saisie à 100% du conteneur parent et empile les labels sur une ligne distincte.

Version mini

Pour avoir une version plus compacte, qui est utile dans les barres d'outils et les espaces restreints, ajoutez l'attribut data-mini="true" à l'élément pour créer une version mini. 

	
<label for="select-choice-min" class="select">Mode de livraison :</label>
<select name="select-choice-min" id="select-choice-min" data-mini="true">
   <option value="standard">Lent : 7 jours</option>
   <option value="rush">Rapide : 3 jours</option>
   <option value="express">Express : le lendemain</option>
   <option value="overnight">De nuit</option>
</select>

Ceci va produire une sélection pas aussi grande que la version standard et disposant d'une taille inférieure.

Conteneurs de champs

Optionnellement, enveloppez-les dans un conteneur avec l'attribut data-role="fieldcontain" afin de les grouper visuellement dans un formulaire plus long.


<div data-role="fieldcontain">
   <label for="select-choice-1" class="select">Mode de livraison :</label>
   <select name="select-choice-1" id="select-choice-1">
      <option value="standard">Lent : 7 jours</option>
      <option value="rush">Rapide : 3 jours</option>
      <option value="express">Express : le lendemain</option>
      <option value="overnight">De nuit</option>
   </select>
</div>

Zones de sélection regroupées verticalement

Pour créer un ensemble groupé de zones de sélection, ajoutez d'abord select et un label correspondant. Réglez l'attribut for de label pour faire correspondre l'id du select ainsi ils sont sémantiquement liés.

Parce que l'élément label sera associé à chaque zonde de sélection, nous recommandons l'enveloppement des sélections dans un élément fieldset qui a une legend qui agit comme un label combiné pour les saisies groupées.

Enfin, on a besoin d'ajouter l'attribut data-role="controlgroup" au fieldset, de sorte qu'il puisse être stylisé comme un groupe.

	
<div data-role="fieldcontain">
    <fieldset data-role="controlgroup">
        <legend>Date de naissance :</legend>
    
        <label for="select-choice-month">Mois</label>
        <select name="select-choice-month" id="select-choice-month">
            <option>Mois</option>
            <option value="jan">Janvier</option>
            <!-- etc. -->
        </select>
    
        <label for="select-choice-day">Jour</label>
        <select name="select-choice-day" id="select-choice-day">
            <option>Jour</option>
            <option value="1">1</option>
            <!-- etc. -->
        </select>
    
        <label for="select-choice-year">Année</label>
        <select name="select-choice-year" id="select-choice-year">
            <option>Année</option>
            <option value="2011">2011</option>
            <!-- etc. -->
        </select>
        
    </fieldset>
</div>

Zones de sélection regroupées horizontalement

Les saisies avec sélection peuvent également être utilisées pour des ensembles groupés avec plus d'une sélection associée. Pour faire un ensemble de bouton horizontal, ajoutez data-type="horizontal" au fieldset. Notez que les boutons qui déclenchent les sélections seront redimensionner en fonction de la valeur de l'option actuellement sélectionnée.

Appel du plugin menu de sélection

Le plugin menu de sélection initialisera automatiquement n'importe quelle page contenant le menu de sélection, il n'y a pas besoin d'un attribut data-role dans le balisage. Cependant, vous pouvez appeler directement le plugin de menu de sélection sur n'importe quel sélecteur, comme n'importe quel plugin normal de jQuery :


$('select').selectmenu();

Menus de sélection personnalisés

Pour des raisons de style avancé, le framework offre également une méthode de génération des menus personnalisés depuis l'existence d'un balisage de menu plutôt que de sélectionner le menu OS natif. Le menu personnalisé supporte les options désactivées et la sélection multiple (alors que le support natif de l'OS du mobile pour les deux est incompatible), cela apporte une façon élégante de gérer les valeurs placeholder et de reconstituer la fonctionnalité manquante sur certaines plateformes telles que le support de optgroup sur Android (tout est expliqué ci-dessous).
