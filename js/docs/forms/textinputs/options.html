Le plugin saisie de texte a les méthodes suivantes :

clearBtn boolean

par défaut : false

Ajoute un bouton d'effacement au saisie (pas disponible pour les zones de texte). Cette option est également définie comme un attribut de données : data-clear-btn="true"

$('.selector').textinput({ clearBtn: true });
clearBtnText string

Par défaut : "clear text"

La description du texte pour le bouton facultatif d'effacement, utile pour les technologies d'assistance comme les lecteurs d'écran. Cette option est également définie comme un attribut de données : data-clear-btn-text="Clear input"

$('.selector').textinput({ clearBtnText: "Clear value" });
disabled boolean

par défaut : false

Définit l'état par défaut de la saisie du texte, à désactiver si la valeur est à "true".

$('.selector').textinput({ disabled: true });
initSelector CSS selector string

default: "input[type='text'], input[type='search'], :jqmData(type='search'), input[type='number'], :jqmData(type='number'), input[type='password'], input[type='email'], input[type='url'], input[type='tel'], textarea, input[type='file'], input:not([type])"

Ceci est utilisé pour définir les sélecteurs (types d'éléments, les rôles de données, etc) qui seront automatiquement initialisé comme des zones de saisie. Pour changer les éléments qui sont initialisés, liez cette option à l'événement mobileinit :

$( document ).on( "mobileinit", function(){
   $.mobile.textinput.prototype.options.initSelector = ".myInputs";
});
mini boolean

par défaut : false

Définit la taille de l'élément pour qu'il soit plus compact : une version mini. Cette option est également définie comme un attribut de données : data-mini="true"

$('.selector').textinput({ mini: true });
preventFocusZoom boolean

Par défaut : true sur les plateformes iOS

Cette option désactive temporairement le zoom de la page quand une saisie personnalisée est actuellement utilisée, ce qui empêche les appareils iOS d'effectuer un zoom de la page dans la saisie. Par défaut, iOS fait souvent un zoom sur les contrôles de formulaire, et le comportement est souvent inutile et intrusif dans les mises en pages optimisés pour le mobile. Cette option est également définie comme un attribut de données : data-prevent-focus-zoom="true"

$('input').textinput({ preventFocusZoom: true });
theme string

par défaut : null, hérité d'un parent

Définit le jeu de couleurs (nuancier) pour toutes les instances de ce widget. Il accepte une seule lettre de a à z qui correspond aux nuanciers inclus dans votre thème. Par défaut, il hérite de la même couleur de nuancier que le conteneur parent qui le possède s'il n'est pas explicitement défini. Cette option est également définie comme un attribut de données : data-theme="a"

$('.selector').textinput({ theme: "a" });
