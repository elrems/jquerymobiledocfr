Les options suivantes s'appliquent à tous les boutons :

corners boolean

Par défaut : true

Applique border-radius au thème du bouton si défini à true. Cette option est également définie comme un attribut de données : data-corners="false"

$('a').buttonMarkup({ corners: false });
Pas de coins arrondis
icon string

par défaut : null

Applique une icône de l'ensemble d'icônes. Cette option est également définie comme un attribut de données : data-icon="star"

$('a').buttonMarkup({ icon: "star" });
Icône étoile
iconpos string

par défaut : "left"

Positions de l'icône dans le bouton. Les valeurs possibles : left, right, top, bottom, none, notext. La valeur notext affichera seulement une icône de bouton sans texte. Cette option est également définie comme un attribut de données : data-iconpos="left"

$('a').buttonMarkup({ iconpos: "right" });
Icône étoile
iconshadow boolean

Par défaut : true

Applique une ombre de thème à l'icône du bouton si défini à true. Cette option est également définie comme un attribut de données : data-iconshadow="false"

$('a').buttonMarkup({ iconshadow: false });
Pas d'ombre sur l'icône
inline boolean

par défaut : null (false)

Si positionné à true, cela rendra le bouton comme un bouton en ligne pour que la largeur soit déterminée par le texte du bouton. Par défaut, c'est null (false) donc le bouton est sur toute la largeur, indépendamment du contenu du texte. Valeurs possibles : true, false. Cette option est également définie comme un attribut de données : data-inline="true"

$('a').buttonMarkup({ inline: true });
Aligné
mini boolean

par défaut : null (false)

Si la valeur est àtrue, cela affichera une version plus compacte du bouton qui utilisera une hauteur moins importante. Valeurs possibles : true, false. Cette option est également définie comme un attribut de données : data-mini="true"

$('a').buttonMarkup({ mini: true });
Mini
shadow boolean

Par défaut : true

Applique le style ombre au bouton si défini à true. Cette option est également définie comme un attribut de données : data-shadow="false"

$('a').buttonMarkup({ shadow: false });
Pas d'ombre du bouton
theme string

par défaut : null, hérité d'un parent

Définit le jeu de couleurs (nuancier) pour toutes les instances de ce widget. Il accepte une seule lettre de a à z qui correspond aux nuanciers inclus dans votre thème. Par défaut, il hérite de la même couleur de nuancier que le conteneur parent qui le possède s'il n'est pas explicitement défini. Cette option est également définie comme un attribut de données : data-theme="a"

$('a').buttonMarkup({ theme: "a" });
Thème A

L'option suivante ne s'applique qu'aux boutons de formulaire, qui sont automatiquement initialisées par le framework :

initSelector CSS selector string

Par défaut : "button, [type='button'], [type='submit'], [type='reset']"

Ceci est utilisé pour définir les sélecteurs (types d'éléments, les rôles de données, etc) qui seront automatiquement initialisé comme des boutons de formulaire. Pour changer les éléments qui sont initialisés, liez cette option à l'événement mobileinit :

$( document ).on( "mobileinit", function(){
   $.mobile.button.prototype.options.initSelector = ".myButtons";
});
