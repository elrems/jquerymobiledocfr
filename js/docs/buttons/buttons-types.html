Les boutons sont codés avec un ancrage HTML standard et des éléments input, puis renforcée par jQuery Mobile pour les rendre plus attractifs et utilisable sur un appareil mobile. Utilisez les liens d'ancrage (éléments a) pour marquer les boutons de navigation et les éléments input ou button pour la soumission de formulaire.

Regardez la référence de l'attribut data- pour voir tous les attributs possibles pour les boutons incluant l'ajout d'icons ou l'affichage inline ou grouped.

Stylisation des liens comme des boutons

Dans le bloc de contenu principal d'une page, vous pouvez styliser tout lien comme un bouton en ajoutant l'attribut data-role="button". Le framework permettra d'améliorer le lien avec un balisage et des classes pour styler le lien comme un bouton. Par exemple, ce balisage :

<a href="index.html" data-role="button">Bouton de lien</a>

Remarque : les liens stylisés comme les boutons qui ont tous les mêmes options visuelles basés sur un bouton de formulaire ci-dessous, mais il y a quelques différences importantes. Les liens basés sur les boutons ne font pas partis du plugin button et sont seulement utilisés par le plugin buttonMarkup pour générer les styles de bouton ainsi les méthodes de bouton de formulaire (enable, disable, refresh) ne sont pas supportés. Si vous avez besoin de désactiver un lien basé sur bouton (ou tout autre élément), il est possible de lui appliquer la classe ui-disabled avec JavaScript afin de réaliser le même effet.

<a href="index.html" data-role="button" class="ui-disabled">Bouton de lien</a>

Version mini

Pour avoir une version plus compacte, qui est utile dans les barres d'outils et les espaces restreints, ajoutez l'attribut data-mini="true" au bouton pour créer une version mini. 

	
<a href="index.html" data-role="button" data-mini="true">Bouton de lien</a>

Ceci va produire un bouton pas aussi grand que la version standard et disposant d'une taille inférieure.

Boutons de formulaire

Pour facilité le style, le framework convertit automatiquement tout élément button ou input avec un type de submit, reset ou button dans un bouton stylisé personnalisé - il n'est pas nécessaire d'ajouter l'attribut data-role="button". Toutefois, si nécessaire, vous pouvez appeler directement le plugin bouton sur n'importe quel sélecteur, comme n'importe quel plugin jQuery :

$('[type="submit"]').button();

Afin de préserver des événements liés au button ou input original, le framework cache l'élément original en le rendant transparent et en le positionnant sur le balisage du nouveau bouton. Quand un utilisateur clique sur le bouton personnalisé, il clique en fait sur l'élément d'origine. Pour empêcher un bouton de formulaire d'être transformé en un bouton amélioré, ajouter l'attribut data-role="none" et le contrôle natif sera rendu.

Bouton basé sur Button :

	
<button>Élément button</button>

Bouton basé sur Input type="button" :

	
<input type="button" value="Button" />

Bouton basé sur Input type="submit" :


<input type="submit" value="Submit Button" />

Bouton basé sur Input type="reset" :


<input type="reset" value="Reset Button" />
