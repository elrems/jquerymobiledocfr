Le mode de table reflow fonctionne en pliant les colonnes de la table dans une présentation empilé qui ressemble à des blocs de label/données pour chaque ligne. Depuis que l'ordre du source HTML d'une table interdit le style d'une table qui ressemble à ceci, le plugin ajoute dynamiquement un peu de balisage pour que l'affichage fonctionne (sans affecter l'accessibilité). Voici une démo d'une table basique utilisant le mode reflow :

Rang Titre Année Classement Commentaires
1 Citizen Kane 1941 100% 74
2 Casablanca 1942 97% 64
3 The Godfather 1972 97% 87
4 Gone with the Wind 1939 96% 87
5 Lawrence of Arabia 1962 94% 87
6 Dr. Strangelove Or How I Learned to Stop Worrying and Love the Bomb 1964 92% 74

Application du mode reflow dans une table

Le mode de table reflow responsive est la plus simple en termes d'exigences de balisage car il ne nécessite qu'une table avec un data-role="table" sur l'élément table. Il n'est pas nécessaire de définir l'attribut data-mode car reflow est la valeur par défaut.

<table data-role="table" id="my-table">

Comment fonctionne le mode reflow

Le plugin fonctionne en analysant les valeurs (ou abbr title) des éléments de la première rangée d'entête (TH). Par exemple, dans le tableau ci-dessus, la troisième entête de la table est analysé pour récupérer le contenu ("Année") :

<th>Année</th>

Le script ajoute alors un élément avec le texte d'entête de la table avant le contenu de chaque cellule de cette colonne. Par exemple, pour chaque cellule du tableau dans la colonne année :

<td>1941</td>

Un élément est ajouté avant le texte de chaque cellule avec une classe de ui-table-cell-label :

<td><b class="ui-table-cell-label">Année</b>1941</td>

Avec notre approche de mobile en premier, les styles de base pour une table reflow empile chaque ligne et chaque cellule présente dans le format de style label/données. Ceci est fait en cachant les lignes d'entête de la table et mettant chaque cellule de table display:block si elles sont empilées. L'élément label injecté dans chaque cellule est stylisé display:inline-block avec la règle min-width:30% pour placer les étiquettes sur la même ligne que le contenu à une largeur uniforme pour former une présentation à 2 colonnes.

Création de table responsive

Par défaut, une table avec le mode reflow affichera le style de présentation empilé sur toutes les largeurs d'écran. Les styles pour rendre la table responsive sont ajoutés en appliquant une media query avec des règles pour passer à la présentation de style tableau au-dessus d'une largeur d'écran spécifique.

Ceci est réalisé en enroulant quelques règles CSS simples et une media query qui s'applique uniquement aux règles en-dessus d'un point d'arrêt d'une certaine largeur. Les styles font apparaitre les lignes d'entête de table, affichent les cellules sous forme de tableau et masquent les éléments label générés au sein de chacune d'elle. Voici un exemple de media query qui change la présentation à 40em (640 pixels) : 

@media ( min-width: 40em ) {
	/* Affiche la ligne entête de la table et définit toutes les cellules à display: table-cell */ 
	.my-custom-breakpoint td,
	.my-custom-breakpoint th,
	.my-custom-breakpoint tbody th,
	.my-custom-breakpoint tbody td,
	.my-custom-breakpoint thead td,
	.my-custom-breakpoint thead th {
		display: table-cell;
		margin: 0;
	}
	/* Masque les labels dans chaque cellule */ 
	.my-custom-breakpoint td .ui-table-cell-label,
	.my-custom-breakpoint th .ui-table-cell-label { 
		display: none;
	}
}

Il est préférable d'utiliser une classe sur la table pour appliquer le point d'arrêt. Ajoutez ces règles à votre feuille de style personnalisée qu'il faut inclure dans le head de la page. Nous vous recommandons de créer un ensemble de classes de point d'arrêt personnalisés qui peut être utilisé pour appliquer des points d'arrêt de table standard dans votre projet.

Dans l'exemple ci-dessus, nous supposons qu'il y a une classe my-custom-breakpoint ajoutée à la table pour appliquer le point d'arrêt. Chacune des règles de media query ont une portée limitée contre la classe de la table qui cible uniquement les tables qui ont la classe my-custom-breakpoint.

Afin que cette technique fonctionne, un navigateur doit prendre en charge les media queries et avoir la capacité de styliser les cellules d'une table comme des éléments en bloc de niveau. En testant, les navigateurs les plus populaires de bureau et de mobile répondent à ces critères, mais les anciennes versions d'Internet Explorer (8 et plus anciennes) reviennent à une présentation de table normale. IE 9 peut prendre en charge cette technique, mais il y a quelques styles supplémentaires nécessaires, ainsi nous recommandons d'appliquer ceux-ci dans un media query max-width pour ne les appliquer seulement en dessous de la présentation de la table, car ils sont difficiles à nier.

Choisir un point d'arrêt

L'objectif est de déterminer la largeur minimale à laquelle l' ensemble de la table s'adapte confortablement à l'intérieur de l'écran. Trouvez cette largeur en remplissant un tableau avec des données réalistes, puis réglez la fenêtre du navigateur jusqu'à ce que la table corresponde complètement et ait un peu d'espace supplémentaire pour tenir compte des différents rendus des appareils. C'est à cet endroit qu'il faut définir le point d'arrêt qui permute entre l'empilement et le mode de présentation en tableau.

Le point d'arrêt de largeur dépend fortement du nombre de colonnes dans la table et le contenu de chaque cellule. Sur certains sites, cela peut être très bas comme 30em (480px) et sur d'autres, cela peut être très large comme 100em (1600 px). Il n'existe aucun moyen pour le framework de décider d'un "point d'arrêt standard" qui fonctionnera pour tout le monde - c'est pourquoi il n'y a pas un point d'arrêt intégré dans la table par défaut.

Nous recommandons l'écriture des largeurs des media queries en em ainsi ils répondent aux changements de taille de police. Pour convertir une largeur en pixel vers du em, divisez la largeur voulue par 16 (pixels). Utilisez cette valeur pour la valeur min-width dans le media query.

L'application d'un point d'arrêt prédéfini

Même si nous vous encourageons vivement à écrire vous-même des points d'arrêt personnalisé, le framework comprend un seul point d'arrêt pré-configuré qui cible le style empilé pour téléphones plus petits et le change pour une présentation en tableau sur les grands téléphones, les tablettes et les appareils de bureau. Pour utiliser ce point d'arrêt prédéfini, ajoutez la classe ui-responsive à la table pour convertir la présentation empilé pour une présentation sous forme de tableau à partir de 560px (35em). Si ce point d'arrêt ne fonctionne pas pour votre contenu, nous vous encourageons à écrire un point d'arrêt personnalisé tel que décrit ci-dessus.

<table data-role="table" class="ui-responsive">

Travailler avec des entêtes de colonnes groupées

Il est assez courant de devoir regrouper logiquement plusieurs colonnes sous une même entête de groupe pour les données financières ou scientifiques. Le framework peut supporter la version la plus simple de ceci, en acceptant deux rangées d'entêtes de table (TH), avec la première rangée contenant un simple attribut colspan pour grouper les colonnes en-dessous. Dans cette configuration, le framework ajoutera une classe à l'étiquette de la première cellule de chaque groupe pour vous permettre de styliser celles-ci différemment et fournir une hiérarchie visuelle supplémentaire.
