Toute page peut être présentée comme une boîte de dialogue modale en ajoutant l'attribut data-rel="dialog" au lien ancre de la page. Lorsque l'attribut "dialog" est appliqué, le framework ajoute des styles pour ajouter des coins arrondis, des marges autour de la page et un fond sombre de façon que le "dialog" semblent être suspendus au-dessus de la page. Par défaut le framework va également ajouter un bouton de fermeture si la boîte de dialogue a une entête.

<a href="foo.html" data-rel="dialog">Ouvrir la boîte de dialogue</a>

Ouvrir la boîte de dialogue

Transitions

Par défaut, la boîte de dialogue sera ouverte avec la transition 'pop'. Comme toutes les pages, vous pouvez spécifier n'importe quelle page de transition que vous voulez sur la boîte de dialogue en ajoutant l'attribut data-transition au lien. Pour mieux ressembler à une boîte de dialogue, nous recommandons de spécifier une transition "pop", "slideup" ou "flip".

<a href="foo.html" data-rel="dialog" data-transition="pop">Ouvrir la boîte de dialogue</a>
data-transition="pop" data-transition="slidedown" data-transition="flip"

Fermeture des boîtes de dialogue

Quand un lien est cliqué dans une boîte de dialogue, le framework fermera automatiquement la boîte de dialogue et la transition vers la page demandée, comme si la boîte de dialogue était une page normale. Néanmoins, les boîtes de dialogue peuvent également être enchaînées, comme expliqué ci-dessous dans "Chainage des boîtes de dialogue". De façon similaire, un lien qui ouvre une fenêtre laissera également la boîte de dialogue en place.

Si la boîte de dialogue a une entête, le framework va ajouter un bouton de fermeture sur le côté gauche de l'entête. Vous pouvez changer la position en ajoutant data-close-btn="right" au conteneur de la boîte de dialogue. Si vous ne voulez pas de bouton de fermeture dans l'entête ou vous voulez ajoutez un bouton de fermeture personnalisé, vous pouvez utiliser data-close-btn="none".

Bouton de fermeture à droite Pas de bouton de fermeture

Pour créer un bouton "Annuler" dans une boîte de dialogue, faites un lien seulement vers la page qui a déclenché la boîte de dialogue pour l'ouvrir et ajouter l'attribut data-rel="back" à votre lien. Ce schéma de liaison à la page précédente est également utilisable tel quel dans des dispositifs non-JS.

Pour les liens générés par JavaScript, vous pouvez simplement définir l'attribut href à "#" et utilisez l'attribut data-rel="back". Vous pouvez aussi appeler la méthode close() de la boîte de dialogue pour fermer par programmation les boîtes de dialogue, par exemple : $('.ui-dialog').dialog('close').

Paramètrage du texte du bouton de fermeture

Tout comme le plugin page, vous pouvez définir le texte du bouton de fermeture d'une boîte de dialogue grâce à l'option ou à l'attribut de données. Cette option est utilisée pour personnaliser le texte du bouton de fermeture qui est utile pour traduire ceci dans différentes langues. Ceci est affiché par défaut comme un bouton d'icône seule, de sorte que le texte n'est pas visible à l'écran, mais est lisible par les lecteurs d'écran et c'est donc une fonctionnalité d'accessibilité importante. L'option peut être configurée pour toutes les boîtes de dialogue en liant l'événement mobileinit et en définissant la propriété $.mobile.dialog.prototype.options.closeBtnText en une chaîne de votre choix, ou vous pouvez placer l'attribut de données data-close-btn-text pour configurer le texte de votre balisage.

Historique & comportement du bouton Retour

Les boîtes de dialogue sont généralement utilisés pour supporter des actions au sein d'une page, le framework n'inclut pas les boîtes de dialogue dans le suivi de l'historique d'état du hash. Cela signifie que les boîtes dialogues n'apparaîtront pas dans la chronologie de votre historique de navigation lorsque le bouton Retour sera cliqué. Par exemple, si vous êtes sur une page, cliquez sur un lien pour ouvrir une boîte de dialogue, fermez la boîte de dialogue, puis naviguer vers une autre page, si vous deviez cliquer sur le bouton Précédent du navigateur, à ce stade, vous reviendrez à la première page, et non pas sur la boîte de dialogue.

Chainage de boite de dialogue

Veuillez notez : Si une boîte de dialogue s'ouvre dans une autre boîte de dialogue (chainage), la fermeture du dernier avec un lien de type data-rel="back" naviguera toujours vers la boîte de dialogue précédente jusqu'à ce que la racine de la page de type data-role="page" soit atteinte. Ceci garantit une navigation cohérente entre les boîtes de dialogue.

Les boîtes de dialogue peuvent être utilisées comme une feuille de contrôle pour offrir de multiples boutons si vous enlever simplement le haut de la marge de l'élément conteneur du dialogue du inner. Par exemple, si votre page de boîte de dialogue a une classe my-dialog, vous pouvez ajouter ce CSS pour épingler la boîte de dialogue en haut : .ui-dialog.my-dialog .ui-dialog-contain { margin-top: 0 }, ou vous pouvez juste appliquer le style à toutes les boîtes de dialogue avec .ui-dialog .ui-dialog-contain { margin-top: 0 }.

Largeur et marges de boîte de dialogue

Par souci de lisibilité, les boîtes de dialogue ont par défaut un width de 92.5% et un max-width de 500 pixels. Il y a aussi un margin pour le haut à 10% pour donner une marge aux boîtes de dialogue plus grande que les écrans, mais se replie en une petite marge sur les smartphones. Si une boîte de dialogue est utilisée comme une feuille de contrôle, le conteneur intérieur de la boîte de dialogue est décalée vers le haut avec 15px pour cacher le style avec les coins (voir ci-dessus). Pour remplacer ces styles, ajoutez le CSS suivant qui écrase les règles de votre feuille de style et peaufine si besoin :


.ui-dialog-contain {
	width: 92.5%;
	max-width: 500px;
	margin: 10% auto 15px auto;
	padding: 0;
	position: relative;
	top: -15px;
}
