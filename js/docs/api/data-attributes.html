Référence des attributs data-

Le framework jQuery Mobile utilise les attributs HTML5 data- pour permettre l'initialisation et la configuration des widgets basés sur le balisage. Ces attributs sont complètement optionnels, en appelant les plugins manuellement et en passant les options directement, cela est également supporté. Pour éviter les conflits de noms avec d'autres plugins ou des frameworks qui utilisent également les attributs data-, définissez un espace de nommage personnalisé en modifiant l'option globale ns.

Bouton

Les liens avec le data-role="button". Les boutons basés sur l'input et les éléments button sont améliorés automatiquement, donc pas besoin de data-role.

data-corners true | false
data-icon home | delete | plus | arrow-u | arrow-d | check | gear | grid | star | custom | arrow-r | arrow-l | minus | refresh | forward | back | alert | info | search
data-iconpos left | right | top | bottom | notext
data-iconshadow true | false
data-inline true | false
data-mini true | false - version d'une taille compacte
data-shadow true | false
data-theme Lettre du nuancier (a-z)

Plusieurs boutons peuvent être enveloppés dans un conteneur avec l'attribut data-role="controlgroup" pour les regrouper verticalement. Ajoutez l'attribut data-type="horizontal" pour que les boutons soient côte à côte. Ajoutez data-mini="true" pour obtenir une version de taille mini/compact pour tous les boutons qui apparaissent dans le controlgroup.

Case à cocher

Des paires de labels et d'inputs avec le type="checkbox" sont automatiquement améliorés, donc pas besoin de data-role

data-mini true | false - version d'une taille compacte
data-role none - Empêche l'auto-amélioration d'utiliser le contrôle natif
data-theme Lettre du nuancier (a-z) - Ajouté à l'élément du formulaire

Plusieurs cases à cocher peuvent être enveloppés dans un conteneur avec l'attribut data-role="controlgroup" pour regrouper verticalement l'ensemble. Ajoutez l'attribut data-type="horizontal" pour que les cases à cocher soit côte à côte. Ajoutez data-mini="true" pour obtenir une version de taille mini/compact pour toutes les cases à cocher qui apparaissent dans le controlgroup.

Controlgroup

Conteneur DIV ou FIELDSET avec data-role="controlgroup". Visuellement intègre de multiples bouton de saisie stylisés en un seul type (cases à cocher, lien basés sur les boutons, radio boutons, sélections) dans un groupe. Pour regrouper les cases à cocher et les radio boutons de formulaire, le conteneur fieldset est recommandé dans un conteneur div avec data-role="fieldcontain" pour améliorer le style des labels.

data-mini true | false - version d'une taille compacte pour tous les éléments d'un controlgroup
data-type horizontal | vertical - Pour l'alignement horizontal ou vertical des éléments

data-corners true | false
data-dom-cache true | false
data-overlay-theme Lettre du nuancier (a-z) - Thème de substitution lorsque la page est ouverte dans une boîte de dialogue
data-theme Lettre du nuancier (a-z)
data-title string - Titre utilisé quand la page est affichée

Amélioration

Conteneur avec data-enhance="false" ou data-ajax="false"

data-enhance true | false
data-ajax true | false

Tous les éléments du DOM au sein d'un conteneur avec data-enhance="false", sauf pour les éléments data-role="page|dialog", seront ignorés lors de l'amélioration initiale et des événements create ultérieurs à condition que le $.mobile.ignoreContentEnabled soit défini avant l'amélioration (par exemple dans un bind de mobileinit).

L'élément d'un lien ou d'un formulaire à l'intérieur des conteneurs data-enhance="false" sera ignoré par la fonctionnalité de navigation du framework lorsque $.mobile.ignoreContentEnabled est défini à true.

Barres d'outils fixe

data-disable-page-zoom true | false - Possibilité de mise à l'échelle pour les pages avec des barres d'outils fixes
data-fullscreen true | false - Paramétrage des barres d'outils pour être au dessus du contenu de la page
data-tap-toggle true | false - Possibilité de basculer la visibilité de la barre d'outils par l'utilisateur (taper/cliquer)
data-transition slide | fade | none - Affiche/Masque la transition lorsqu'on tape/clique
data-update-page-padding true | false - Avoir la mise à jour de l'espacement du haut et du bas de la page sur les événements resize, transition, "updatelayout" (le framework met toujours à jour sur l'événement "pageshow").
data-visible-on-page-show true | false - Visibilité de la barre d'outils lorsque la page parent est affichée

Barre de navigation

Un nombre de LIs enveloppés dans un conteneur avec data-role="navbar"

data-iconpos left | right | top | bottom | notext

Pour ajouter des icônes aux éléments de la barre de navigation, l'attribut data-icon est utilisé, spécifiant une icône mobile standard pour chaque élément.

Les barres de navigation héritent du nuancier de thème de leur conteneur parent. Le paramétrage de l'attribut data-theme à une barre de navigation n'est pas supporté. L'attribut data-theme peut être défini individuellement aux liens à l'intérieur de la barre de navigation.
