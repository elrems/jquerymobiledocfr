Thématisation aperçu

Le système de thématisation utilisé dans jQuery Mobile est similaire au système Themeroller de JQuery UI avec quelques améliorations importantes :

  • Il tire parti des propriétés CSS3 pour ajouter des coins arrondis, des ombres aux boites et aux textes et des gradients à la place des images, permettant que le fichier de thème soit très léger et en réduisant les requêtes au serveur.
  • Les thèmes comprennent plusieurs "nuanciers" de couleur - composés chacun d'une barre d'entête, d'un corps de contenu et les états du bouton qui peuvent être librement mélangés et combinés pour créer la texture visuelle - pour rendre le design le plus riche possible.
  • La thématisation ouverte permet jusqu'à 26 nuanciers uniques par thème, pour ajouter de la variété quasi illimitée aux conceptions.
  • Tous les contextes utilisent maintenant les gradients CSS3 pour réduire considérablement la taille du fichier et le nombre de requêtes au serveur.
  • Il y a un ensemble d'icône simplifiée dans un sprite pour réduire le poids de l'image.

ThemeRoller

La meilleure façon de créer des thèmes personnalisés, c'est avec l'outil Themeroller. Il vous permet de créer un thème, puis de télécharger un fichier CSS personnalisé qui est prêt à être déposé dans votre projet.

Correspondance de nuancier de thème par défaut pour les composants

Si aucune lettre de nuancier de thème est défini pour tous, le framework utilise le nuancier "a" (noir dans le thème par défaut) pour les entêtes et les pieds de page et le "c" (gris clair dans le thème par défaut) pour le contenu de la page pour optimiser le contraste entre les deux.

Tous les éléments dans les conteneurs héritent du nuancier de leur parent. Les exceptions à cette règle sont les séparateurs de liste dans les listes, l'entête de la liste des pages imbriquées, et le bouton dans les listes séparées avec un bouton. Pour ces derniers, c'est par défaut à "b" (en bleu sur le thème par défaut). Les bulles de compteur sont par défaut à "c" (argent dans le thème par défaut).

Notez qu'il y a aussi le nuancier nommé "active" (bleu vif dans le thème par défaut) qui est utilisé pour indiquer un élément sélectionné actif. Voir l'état global "actif" plus loin dans cette page pour plus d'informations sur le nuancier active.

La boîte de dialogue de chargement des pages et le message d'erreur n'héritons pas du nuancier du thème. Le chargement de la boîte de dialogue a par défaut le nuancier "a" (noir dans le thème par défaut) et le message d'erreur à le nuancier "e" (en jaune sur le thème par défaut) Vous pouvez configurer les paramètres par défaut globalement. Voir configuration par défaut.

Thèmes & nuanciers

Le système de thème sépare la couleur et la texture, grâce à des styles structuraux qui définissent les choses comme la marge et les dimensions. Cela permet que les textures et les couleurs soient définies une fois dans la feuille de style et soient mélangées, appariées et combinées pour obtenir un large éventail d'effets visuels.

Chaque thème comprend plusieurs paramètres globaux, y compris la famille de police, les ombres portées pour les superpositions et les valeurs du rayon du coin pour les boutons et les boîtes. En outre, le thème peut inclure plusieurs "nuanciers" de couleur, chacun avec des valeurs de couleur pour les barres, les blocs de contenu, les éléments de liste et de bouton et la police text-shadow.

Le thème par défaut comprend 5 nuanciers qui sont donnés par des lettres (a, b, c, d, e) pour une référence rapide. Pour faire la cartographie des nuanciers de couleur uniforme dans nos widgets, nous avons suivi la convention que le nuancier "a" est le plus haut niveau de priorité visuelle (noir dans notre thème par défaut), "b" est de niveau secondaire (bleu) et "c" est le niveau de référence (gris) que nous utilisons par défaut dans de nombreuses situations, "d" pour un autre niveau secondaire et "e" comme un nuancier pour mettre en valeur. Les thèmes peuvent avoir des nuanciers supplémentaires pour des couleurs de mise en valeur ou des situations spécifiques. Par exemple, vous pouvez ajouter un nouveau nuancier "f" qui a une barre et un bouton rouge pour une utilisation dans des situations d'erreur.

La plupart des changements de thème peuvent être fait en utilisant Themeroller, mais c'est aussi simple de modifier manuellement les nuanciers de base dans le thème par défaut et/ ou d'ajouter des nuanciers supplémentaires en éditant le fichier css du thème. Il suffit de copier un bloc de styles swatch, renommer les classes avec le nom de la nouvelle lettre du nuancier et de régler les couleurs comme bon vous semble.

État "Active" globale

Le framework jQuery Mobile utilise un nuancier de thème appelé "active" (bleu clair dans le thème par défaut) pour constamment indiquer l'état sélectionné, indépendamment du nuancier individuel du widget donné. Nous appliquons ceci dans les contrôles de navigation et de formulaire chaque fois qu'il y a un besoin d'indiquer ce qui est actuellement sélectionné. Parce que ce nuancier de thème est conçu pour un retour utilisateur clair et cohérent, il ne peut pas être remplacé via le balisage, il est fixé une fois dans le thème et appliqué par le framework à chaque fois qu'un état sélectionné ou actif est nécessaire. Le style de cet état est dans la feuille de style du thème dans les règles de style ui-btn-actif.

Icônes

Il y a un ensemble de base d'icônes standards inclus dans le framework qui peut être attribué à n'importe quel bouton. Afin de minimiser la taille de téléchargement des icônes de base, jQuery Mobile inclut seulement ces icônes en blanc et ajoute automatiquement un cercle semi-transparent noir derrière l'icône pour s'assurer qu'il a un bon contraste sur toutes les couleurs de fond.

Classes de thème

Affectation des nuanciers de couleurs à travers l'attribut data-theme est une façon d'exploiter le système de thème, mais il est également possible d'appliquer n'importe quel nuancier de thème directement sur votre balisage à travers des classes pour appliquer les couleurs, les textures et le formatage de police de votre thème à n'importe quel balisage. Ceci est particulièrement utile lorsque vous créez votre propre mise en page personnalisée des éléments ou des widgets UI. Voici quelques classes de thème commune, mais plusieurs sont disponibles dans la feuille de style de thème :

ui-bar-(a-z)
Applique les styles de thème de barre d'outils pour la lettre du nuancier sélectionné. Communément utilisé en conjonction avec la classe structurelle ui-bar pour ajouter les styles standards d'espacement.
ui-body-(a-z)
Applique les styles de thème de contenu du corps pour la lettre du nuancier sélectionné. Communément utilisé en conjonction avec la classe structurelle ui-body pour ajouter les styles standards d'espacement de bloc de contenu.
ui-btn-up-(a-z)
Applique les styles de thème d'élément bouton/cliquable pour la lettre du nuancier sélectionné. Couramment utilisés avec les statuts de classe d'interaction ui-btn-hover-(a-z) et ui-btn-down-(a-z) pour fournir un retour visuelle et ui-btn-active pour indiquer les statuts "on" ou sélectionnés.
ui-corner-all
Applique border-radius sur le thème globale pour les coins arrondis et est utilisé pour les conteneurs ou les éléments regroupés dans le le framework (listes insérées, ensembles de radio bouton). Il y a des classes supplémentaires pour toutes les combinaisons possibles de coin arrondi, par exemple : ui-corner-tl (Haut gauche uniquement), -top (Deux coins supérieurs), -left (Deux coins à gauche), etc. Une deuxième série complète de classes d'angle est fourni pour les boutons de sorte que ces derniers puissent avoir un arrondi différent. L'utilisation de ces classes ont une convention de nommage similaire, mais avec "btn-corner" au lieu de "corner", comme ceci : .ui-btn-corner-all.
ui-shadow
Applique le style ombre du thème globale sur tous éléments en utilisant la propriété CSS box-shadow.
ui-disabled
Applique le look désactivé qui réduit essentiellement l'opacité d'un élément quelconque de cette classe à 30%, cache le curseur, et définit pointer-events: none; ce qui empêche toute interaction dans de nombreux navigateurs modernes.

Surcharge des thèmes

Les thèmes sont conçues comme un solide point de départ, mais sont destinés à être personnalisés. Puisque tout est contrôlé par CSS, il est facile d'utiliser un outil Web d'inspection pour identifier les propriétés de style que vous voulez modifier. L'ensemble des classes de thème (global) et des classes structurelles sémantique (widget-specific) ajoutés aux éléments fournissent un ensemble riche de sélecteurs possible pour cibler les remplacements de style. Nous vous recommandons d'ajouter une feuille de style externe à head, placée après la structure et le style de références thématiques, qui contiennent tous vos remplacements de style. Cela vous permet de facilement de mettre à jour les nouvelles versions de la bibliothèque parce les remplacements sont séparés du code de la bibliothèque.
