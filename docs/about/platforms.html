jQuery Mobile plateformes prises en charge

jQuery Mobile a un large support pour la grande majorité de tous les bureaux modernes, smartphone, tablette, et les plateformes e-reader. En plus, les téléphones polyvalents et les navigateurs plus anciens sont également pris en charge en raison de notre démarche d'amélioration progressive. Nous sommes très fiers de notre engagement envers l'accessibilité universelle à travers notre large support pour toutes les plateformes populaires.

Nous utilisons un système de support des plateformes gradué à 3 niveaux : A (complet), B (complet sans l'Ajax), C (HTML de base) La fidélité visuelle de l'expérience et de la fluidité des transitions de page est très dépendante des capacités du rendu CSS de l'appareil et de la plateforme, donc tous les grades A n'auront pas une qualité au pixel près, mais c'est la nature d'internet.

* Remarque : si jQuery 1.8+ est utilisé avec jQuery Mobile, iOS 3.x et BB5 sont repositionnés vers le support grade C car le cœur de jQuery 1.8+ a abandonné le support de méthode que ces plateformes ont besoin pour une fonctionnalité complète.

Grade A - Full expérience accrue avec l'Ajax basé sur des transitions de page animés.

  • Apple iOS 3.2*-6.0 : testé sur l'iPad d'origine (4.3 / 5.0), l'iPad 2 (4.3), l'iPad 3 (5.1 / 6.0), l'iPhone d'origine (3.1), l'iPhone 3 (3.2), 3GS (4.3), 4 (4.3 / 5.0) et 4S (5.1 / 6.0)
  • Android 2.1-2.3 : testé sur l'HTC Incredible (2.2), Droid d'origine (2.2), HTC Aria (2.1), Google Nexus S (2.3). Fonctionnel sur 1.5 & 1.6 mais la performance peut être paresseuse, testé sur Google G1 (1.5)
  • Android 3.2 Honeycomb - testé sur Samsung Galaxy Tab 10.1 et Motorola XOOM
  • Android 4.0 (ICS)  - Testé sur le Galaxy Nexus. Remarque : la performance des transitions peut être faible sur les appareils mis à jour
  • Android 4.1 (Jelly Bean)  - Testé sur le Galaxy Nexus et Galaxy 7.
  • Windows Phone 7-7.5 : testé sur HTC Surround (7.0) HTC Trophy (7.5), LG-E900 (7.5) et Nokia Lumia 800
  • Blackberry 6.0 : testé sur Torch 9800 et Style 9670
  • Blackberry 7 : testé sur BlackBerry Torch 9810
  • Blackberry Playbook (1.0-2.0) : testé sur PlayBook
  • Palm WebOS (1.4-2.0) : testé sur Palm Pixi (1.4), Pre (1.4), Pre 2 (2.0)
  • Palm WebOS 3.0 - testé sur HP TouchPad
  • Firefox Mobile 15 - testé sur appareil Android 2.3 et 4.1
  • Chrome pour Android 18 : testé sur appareil Android 4.0 et 4.1
  • Skyfire 4.1 : testé sur appareil Android 2.3
  • Opera Mobile 11.5-12 : testé sur Android 2.3
  • Meego 1.2 : testé sur Nokia N950 et N9
  • Tizen (pré-version) : testé sur matériel précoce
  • Samsung Bada 2.0 - testé sur Samsung Wave 3, navigateur Dolphin
  • UC Browser : testé sur appareil Android 2.3
  • Kindle 3 et Fire : testé pour chacun sur le navigateur intégré WebKit
  • Nook Color 1.4.1 : testé sur la tablette d'origine Nook Color, c'est pas le cas de la tablette Nook
  • Chrome de bureau 11-21 : testé sur OS X 10.7 et Windows 7
  • Safari de bureau 4-5 : testé sur OS X 10.7 et Windows 7
  • Firefox de bureau 4-15 : testé sur OS X 10.7 et Windows 7
  • Internet Explorer 7-10 - testé sur Windows XP, Vista et 7
  • Opera de bureau 10-12 - testé sur OS X 10.7 et Windows 7

grade B - une expérience améliorée à l'exception des fonctions de navigation Ajax.

  • Blackberry 5.0* : testé sur Storm 2 9550, Bold 9770
  • Opera Mini 7 : testé sur iOS 5.1 et Android 2.3
  • Nokia Symbian^3 - testé sur Nokia N8 (Symbian^3), C7 (Symbian^3), en cours sur N97 (Symbian^1)

grade C - de base, l'expérience de HTML non-améliorée qui est toujours fonctionnelle

  • Blackberry4.x : testé sur Curve 8330
  • Windows Mobile : testé sur HTC Leo (WInMo 5.2)
  • Toutes les anciennes plateformes de smartphone et de téléphone - Tout appareil qui ne supporte pas les requêtes des médias recevront le basique, expérience de grade C
