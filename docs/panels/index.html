Où va le balisage du panneau dans la page

Les panneaux sont conçus pour être aussi flexible que possible afin de simplifier la création des menus, des colonnes pliables, des tiroirs, des panneaux inspecteurs et plus encore.

Un panneau doit être rattaché aux éléments header, content et footer à l'intérieur de la page jQuery Mobile. Vous pouvez ajouter le balisage du panneau soit avant ou après ces éléments, mais pas entre les deux. Un panneau ne peut pas être placé à l'extérieur d'une page, mais cette contrainte sera supprimée dans une future version.

Voici un exemple du panneau avant le header, content et footer dans l'ordre de la source :


<div data-role="page">			

    <div data-role="panel" id="mypanel">
        <!-- le contenu du panneau se place ici -->
    </div><!-- /panel -->

    <!-- header -->
    <!-- content -->
    <!-- footer -->

</div><!-- page -->

Alternativement, il est possible d'ajouter le balisage du panneau after le header, content et footer dans l'ordre de la source, juste avant la fin du conteneur de page. Selon l'endroit où vous placez le balisage du panneau, dépendra de la façon dont vous voulez que contenu de la page soit lu pour les personnes qui se trouvent sur la page dans un dispositif de grade C (HTML seulement) ou d'un lecteur d'écran.

Conventions de balisage du panneau

Un panneau est constitué d'un conteneur avec l'attribut data-role="panel" et un ID unique. Cet ID sera référencé par un lien ou un bouton pour ouvrir et fermer le panneau. Le balisage du panneau le plus élémentaire ressemble à ceci :


<div data-role="panel" id="mypanel">
    <!-- le contenu du panneau se place ici -->
</div>

La position du panneau sur l'écran est définit par l'attribut data-position. Par défaut, c'est left, ce qui signifie qu'il apparaîtra à partir du bord gauche de l'écran. Spécifiez data-position="right" pour le faire apparaître à partir du bord droit.

Le mode d'affichage du panneau est définit par l'attribut data-display. Par défaut, c'est reveal, ce qui signifie que le panneau s'ouvrira à côté de la page et poussera la page plus loin. Spécifiez data-display="overlay" pour le faire apparaitre au dessus du contenu de la page. Un troisième mode, data-display="push" anime à la fois le panneau et la page.

Voici un exemple d'un panneau avec une position personnalisée et une option d'affichage de définie :


<div data-role="panel" id="mypanel" data-position="right" data-display="push">
    <!-- le contenu du panneau se place ici -->
</div>

Ouverture d'un panneau

L'aperçu du panneau est déclenché par un lien quelque part sur ​​la page ou en appelant directement la méthode open du panneau. Les valeurs par défaut placent le panneau sur la gauche et pousse la page quand il s'ouvre. Ouvrez un panneau par programmation ainsi :

$( "#idofpanel" ).panel( "open" , optionsHash );

Pour contrôler un panneau depuis un lien, pointez le href à la référence du ID du panneau que vous voulez faire apparaitre (mypanel dans l'exemple ci-dessus). Ceci indique au framework de lier le lien au panneau. Ce lien permet de basculer l'affichage de la popup afin d'ouvrir le panneau en cliquant dessus et de le fermer en cliquant à nouveau dessus.


<a href="#mypanel">Ouvrir le panneau</a>
Panneau par défaut

Lorsque vous utilisez des balises pour contrôler les panneaux, vous ne pouvez avoir à la fois qu'un seul panneau ouvert. En cliquant sur un lien pour ouvrir un deuxième panneau alors qu'un panneau est déjà ouvert, cela fermera automatiquement le premier. Ceci est fait pour maintenir la simplicité d'une configuration de balisage unique.

Cela dit, il est possible d'ouvrir plusieurs panneaux à la fois par programmation :


$( "#idofpanel" )
    .panel( "open" , optionsHash )
    .then( function( options ){
        $( "#idofpanel2" ).panel( "open" , options );
    });

Fermeture d'un panneau

En cliquant sur ​​le lien qui a ouvert le panneau, en basculant à droite ou à gauche ou en tapant sur ​​la touche Echap, cela ferme le panneau. Pour désactiver le comportement de fermeture par le swipe, ajoutez l'attribut data-swipe-close="false" au panneau.

Par défaut, les panneaux peuvent être fermés soit en cliquant en dehors du widget panneau. Pour éviter cela, ajoutez l'attribut data-dismissible="false" au panneau. Il est possible d'avoir le panneau et la page côte à côte sur des largeurs d'écran plus larges et d'empêcher le comportement du clic en dehors du panneau qu'à partir d'une certaine largeur d'écran en appliquant une media query. Voir la section responsive ci-dessous pour plus de détails.

Un panneau peut également être fermé en appelant directement la méthode closedu panneau.

$( "#idofpanel" ).panel( "close" );

Il est fréquent d'ajouter un bouton de fermeture à l'intérieur du panneau. Pour ajouter le lien qui fermera le panneau, ajoutez l'attribut data-rel="close" pour dire au framework de le fermer quand il est cliqué. Il est important de s'assurer que ce lien est également disponible si JavaScript n'est pas disponible, c'est pourquoi nous recommandons que le href pointe sur l'ID de la page où l'utilisateur doit aller à la fermeture. Par exemple, si le bouton pour ouvrir le panneau est dans la barre d'entête qui a un ID my-header, le lien de fermeture dans le panneau doit être :


<a href="#my-header" data-rel="close">Fermer le panneau</a>

Animations du panneau

Les panneaux s'animeront si le navigateur prend en charge les transformations 3D, c'est les mêmes critères pour le support d'animation CSS que nous utilisons pour les transitions de page. Les animations du panneau utilisent la transformation CSS translate3d(x,y,z) pour s'assurer qu'ils sont fluides.

Le framework a une fonctionnalité pour détecter si les propriétés CSS requises sont soutenues et se rabattra sur un simple masquer/afficher s'il ne sont pas disponibles. Après des tests approfondis, nous avons décidé de ne pas animer les panneaux sur les plateformes les moins compétentes parce que les animations n'étaient pas plus appropriées qu'un simple masquer/afficher.

L'option animate permet de désactiver les animations du panneau pour tous les appareils. Pour désactiver les animations via le balisage, ajoutez l'attribut data-animate="false" au conteneur du panneau.

Positionnement du panneau

Le panneau sera affiché avec la propriété CSS position:absolute, cela permet de le faire défiler avec la page. Quand un panneau est ouvert, le framework vérifie si le bas du contenu du panneau est visible et, si ce n'est pas le cas, il défile vers le haut de la page.

Vous pouvez définir un panneau avec position:fixed, ainsi son contenu apparaîtra peu importe la distance au bas de la page que vous défiler, en ajoutant l'attribut data-position-fixed="true" au panneau. Le framework vérifie également si le contenu du panneau s'adapte à l'intérieur de la fenêtre avant d'appliquer le positionnement fixe car cette propriété empêche le contenu du panneau de défiler et en utilisant overflow qui n'est pas bien supporté pour être utiliser à ce moment . Si le contenu du panneau est trop long pour tenir dans la fenêtre, le framework affichera tout simplement le panneau sans positionnement absolu.

En général, nous vous recommandons de placer les boutons qui ouvrent le panneau tout en haut de l'écran, c'est le modèle d'interface utilisateur le plus courant pour les panneaux. Ceci évitera le besoin de défiler et rendra également les transitions un peu plus lisses.

Notez qu'il y a des problèmes avec le positionnement fixe dans le WebView des applications d'Android (pas le navigateur) qui peuvent causer des problèmes de mise en page. Nous vous recommandons de ne pas utiliser l'option de positionnement fixe du panneau dans le cas d'un déploiement d'une application Android. En outre, bien que le framework supporte la combinaison de panneaux et de barres d'outils fixe, les barres d'outils fixées n'ont pas de transition avec le contenu de la page.

Style des panneaux

Par défaut, les panneaux ont des styles très simples, ce qui vous permet de personnaliser selon les besoins. Les panneaux sont essentiellement de simples blocs, sans marge, qui se positionnent de chaque côté du contenu de la page. Le framework enveloppe le contenu du panneau dans un div avec la classe ui-pannel-inner qui a un espacement de 15 pixels. Si nécessaire, vous pouvez remplacer ceci avec un CSS personnalisé ou utiliser l'option classes.panelInner pour définir un nom de classe différent pour le div

Les panneaux ont une largeur fixe de 17em (272 pixels), ce qui est assez étroit pour montrer encore une partie du contenu de la page, cela rend plus facile la possibilité de fermer en cliquant en dehors du panneau, et cela a toujours l'air bien sur un grand écran de tablette ou de bureau. Les styles pour définir la largeur des panneaux sont assez complexes, mais ceux-ci peuvent être remplacées avec des CSS selon les besoins.

A part la largeur et la hauteur à 100%, les panneaux sont peu stylisés. Vous pouvez définir un thème sur le panneau par l'ajout de data-theme sur le conteneur du panneau ou selon les besoins par l'ajout de vos propres classes.

Notez que l'ajout d'espacement, de bordure ou de marge directement dans le conteneur du panneau, modifie les dimensions globales et cela peut affecté le positionnement et l'animation.

Pour ajouter de l'espace à l'intérieur d'un panneau, nous recommandons d'ajouter un conteneur à l'intérieur du panneau et d'appliquer les styles à celui-ci afin d'éviter tout problème. Tous les exemples de cette page suivent ce modèle.

Création de panneau responsive

Lorsque l'affichage push ou reveal est utilisé, le panneau pousse la page sur le côté lorsqu'il s'ouvre. Étant donné qu'une partie de la page est poussée hors de l'écran, le panneau est modale et doit être fermé pour interagir à nouveau avec le contenu de la page. Sur les grands écrans, vous voudrez peut-être que le panneau fonctionne plus comme une colonne télescopique qui peut être ouvert et utilisé à côté de la page pour une meilleure utilisation de l'écran.

Pour que la page fonctionne à côté du panneau ouvert, il faut la redimensionner avec une largeur qui s'adapte à celle du panneau. Cela peut être fait uniquement avec des CSS en ajoutant une marge à gauche ou à droite égale à la largeur du panneau (17em) pour le contenu de la page pour forcer un redimensionnement. Deuxièmement, la couche invisible placée sur la page, pour le clic permettant de faire disparaitre le panneau, est cachée avec le CSS de sorte que vous puissiez cliquer sur la page et ne pas fermer le panneau.

Voici un exemple de ces règles enveloppés dans une media query qui applique uniquement ce comportement au dessus de 35em (560px) :


/* wrap push on wide viewports once open */
@media (min-width: 35em){
	.ui-responsive-panel.ui-page-panel-open .ui-panel-content-fixed-toolbar-open.ui-panel-content-fixed-toolbar-display-push,
	.ui-responsive-panel.ui-page-panel-open .ui-panel-content-fixed-toolbar-open.ui-panel-content-fixed-toolbar-display-reveal,
	.ui-responsive-panel.ui-page-panel-open .ui-panel-content-wrap-open.ui-panel-content-wrap-display-push,
	.ui-responsive-panel.ui-page-panel-open .ui-panel-content-wrap-open.ui-panel-content-wrap-display-reveal {
		margin-right: 17em;
	}
	.ui-responsive-panel.ui-page-panel-open .ui-panel-content-fixed-toolbar-open.ui-panel-content-fixed-toolbar-display-push.ui-panel-content-fixed-toolbar-position-right,
	.ui-responsive-panel.ui-page-panel-open .ui-panel-content-fixed-toolbar-open.ui-panel-content-fixed-toolbar-display-reveal.ui-panel-content-fixed-toolbar-position-right,
	.ui-responsive-panel.ui-page-panel-open .ui-panel-content-wrap-open.ui-panel-content-wrap-display-push.ui-panel-content-wrap-position-right,
	.ui-responsive-panel.ui-page-panel-open .ui-panel-content-wrap-open.ui-panel-content-wrap-display-reveal.ui-panel-content-wrap-position-right {
		margin: 0 0 0 17em;
	}
	.ui-responsive-panel.ui-page-panel-open .ui-panel-content-fixed-toolbar-open.ui-panel-content-fixed-toolbar-display-push,
	.ui-responsive-panel.ui-page-panel-open .ui-panel-content-fixed-toolbar-open.ui-panel-content-fixed-toolbar-display-reveal {
		width: auto;	
	}
	.ui-responsive-panel .ui-panel-dismiss-display-push {
		display: none;
	}
}

L'application d'un point d'arrêt prédéfini

Incluez dans le style du widget un point d'arrêt prédéfini pour que ce comportement se déclenche au 55em (880px). Ce point d'arrêt n'est pas appliquée par défaut afin de rendre plus facile l'écriture des points d'arrêt personnalisées qui fonctionnent mieux pour votre contenu et votre design. Pour appliquer le point d'arrêt prédéfini, ajoutez la classe ui-responsive-panel au conteneur de la page (pas au panneau).
