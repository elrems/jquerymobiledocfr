Le plugin popup a les événements personnalisés suivants :

popupbeforeposition déclenchée après qu'une popup ait achevé les préparatifs pour l'ouverture, mais n'a pas encore été ouvert

$( ".selector" ).on({
   popupbeforeposition: function(event, ui) { ... }
});

Cet événement est déclenché lorsque la popup a achevé les préparatifs pour apparaître sur l'écran. À ce stade, la popup n'a pas commencé les animations et il n'a pas encore calculé les coordonnées où il apparaîtra à l'écran. La gestion de cet événement donne l'occasion de modifier le contenu de la popup avant qu'elle n'apparaisse sur l'écran. Par exemple, le contenu peut être réduit ou certaines parties peuvent être masquées ou supprimées si elles sont trop larges ou trop grandes.

popupafteropendéclenché lorsque la popup est complètement ouverte

$( ".selector" ).on({
   popupafteropen: function(event, ui) { ... }
});

Cet événement est déclenché après que le popup soit complètement affiché sur l'écran, ce qui signifie que toutes les animations associées soient terminées.

popupafterclose déclenché lorsque la popup est complètement fermée

$( ".selector" ).on({
   popupafterclose: function(event, ui) { ... }
});

Cet événement est déclenché après que le popup soit complètement masqué de l'écran, ce qui signifie que toutes les animations associées soient terminées.
