Le plugin dialog a les options suivantes :

closeBtn string

par défaut : "left"

Définit la position du bouton de fermeture de la boîte de dialogue dans l'entête (left ou right) ou évite que le framework ajoute un bouton de fermeture (none).

Cette option est également définie comme un attribut de données : data-close-btn.

$( ".selector" ).dialog({ closeBtn: "none" });
closeBtnText string

Par défaut : "Close"

Personnalise le texte du bouton de fermeture, ceci est utile pour le traduire en différentes langues. Ceci est affiché par défaut comme un bouton d'icône seule, de sorte que le texte n'est pas visible à l'écran, mais est lisible par les lecteurs d'écran et c'est donc une fonctionnalité d'accessibilité importante.

Cette option est également définie comme un attribut de données : data-close-btn-text.

$( ".selector" ).dialog({ closeBtnText: "Fermer" });
corners boolean

Par défaut : true

Définit s'il faut dessiner la boite de dialogue avec des coins arrondis.

Cette option est également définie comme un attribut de données : data-corners="false".

$( ".selector" ).dialog({ corners: false });
initSelector CSS selector string

Par défaut : ":jqmData(role='dialog')"

This is used to define the selectors (element types, data roles, etc.) that will automatically be initialized as dialogs. Pour changer les éléments qui sont initialisés, liez cette option à l'événement mobileinit :

$( document ).on( "mobileinit", function(){
   $.mobile.dialog.prototype.options.initSelector = ".mydialog";
});
overlayTheme string

Par défaut : "a"

Les boîtes de dialogue apparaissent en flottant au dessus d'une couche superposée. Cette superposition adopte la couleur de contenu du nuancier "a" par défaut, mais l'attribut data-overlay-theme peut être ajouté à l'enveloppe de la page pour définir la superposition avec une lettre du nuancier.

$( ".selector" ).dialog({ overlayTheme: "e" });

Cette option est également définie comme un attribut de données : data-overlay-theme="e"
