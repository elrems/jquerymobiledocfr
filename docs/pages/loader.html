Le Widget de chargement de Page

La boîte de dialogue de chargement des pages a été préalablement configuré globalement avec trois paramètres $.mobile.loadingMessage, $.mobile.loadingMessageTextVisible et $.mobile.loadingMessageTheme qui sont maintenant obsolètes. De plus les méthodes d'affichage et de masquage, $.mobile.showPageLoadingMsg et $.mobile.hidePageLoadingMsg sont aussi obsolètes.

Le widget de chargement de la page gère la tâche pour afficher la boîte de dialogue de chargement lorsque jQuery Mobile met du contenu via Ajax. Il peut également être affiché manuellement par les actions de chargement personnalisées en utilisant la méthode $.mobile.loading (Voir les docs de méthode globale).

Pour configurer la boîte de dialogue de chargement globalement, les paramètres suivants peuvent être définis sur son prototype au cours de l'événement mobileinit.

Ci-dessous les valeurs par défaut : 
          
$( document ).on( 'mobileinit', function(){
  $.mobile.loader.prototype.options.text = "loading";
  $.mobile.loader.prototype.options.textVisible = false;
  $.mobile.loader.prototype.options.theme = "a";
  $.mobile.loader.prototype.options.html = "";
});

Ces paramètres seront remplacés par les paramètres obsolètes afin de préserver la compatibilité descendante. Une fois que les paramètres obsolètes sont retirés, ces valeurs par défaut seront uniquement remplacée par l'objet params décrit dans les docs de méthode globale sous $.mobile.loading.

          
$.mobile.loading( 'show', {
	text: 'foo',
	textVisible: true,
	theme: 'z',
	html: ""
});
